非兼容性变化

core：

1.对于封闭资源来说：gettype（）函数返回“resource（closed）”,代替了之前的“unknown type”。
2.对于__PHP_Incomplete_Class
类对象，is_object()函数将返回true。
3.对Netware操作系统的支持已废弃；
4.array转换成object((object)或者settype（）函数)将把整型键转换成属性名。这修复了之前的的数字键不能成为属性名的问题。
4.object类型转换成array时（（array）或者settype（）函数），以及对象的属性映射到数组中时（get_object_vars()），会转换整数属性到数字键。这修复了之前的整数属性不可访问的问题。
5.对未定义常量的非限定性引用将导致一个waring级别的错误，代替了之前的notice，未来版本会是error级别的错误。
6.最低支持的Windows版本是Windows 7和Windows Server 2008 R2。
7.原生的trait属性值的兼容性检查将不再进行。
8.”object”将不能被用作类名。

BcMath

1.和fmod（）函数一样，bcmod（）函数将不再把数值的小数部分截断变成整数，代替了之前的取模（%）操作。举个例子，bcmod(‘4’,’3.5’)现在将返回0.5而不是1。

Hash

hash_hmac(), hash_hmac_file(), hash_pbkdf2() 和hash_init()（用HASH_HMAC）这些函数将不再支持非加密的哈希函数。

JSON

json_decode()函数的第二个参数assoc如果是null的话，options支持选项中的JSON_OBJECT_AS_ARRAY将被被派上用场，在之前的版本中这个参数是被我们一直忽略的。

Session

1.移除了register_globals相关的代码，”!”可以被用在$_SESSION的键名中。
2.session被用于管理会话状态,禁止无效操作，当你的代码无效时将会触发该变化。如果你遇到了这个问题，那么这意味这你的代码有问题。
2.1sesseion_start(),session_status(),session_regenerate_id()函数被用于设置或者返回正确的会话状态。
2.2函数返回bool值从null。这些函数如果可以有空的参数，并且空的参数将被检查。 session_unset(), session_write_close()/session_commit(), session_abort()，session_reset()
2.3函数禁止无效的会话状态和http头状态。返回正确的布尔值和value值。
2.4ini的值的设定需通过ini_set（）函数的成功更新。无效的ini修改操作将不会成功。
2.5由于会话被正确的管理，部分E_ERROR级别的信息将转化成E_WARNING级别。
2.6会话将不再被无效的和无用的会话初始化
2.7当头部信息已经被发送，此时去设置新的ini值时将不再工作。老的php将接收新的值即使新的值并非是有效的。
现在这些正确的行为可能会影响到cli命令行模式下管理会话的甲苯。使用输出缓冲，就像是在网页应用那样将解决在cli脚本下的问题。
-standard
在64位机器上，mt_rand（）函数和rand（）函数可能会不一样，自7.1版本以来。
